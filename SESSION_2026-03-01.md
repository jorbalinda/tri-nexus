### Security Audit and Hardening
A full audit was run using the CLAUDE.md security context file. The following was fixed:

**Security Headers**
- `next.config.ts` was essentially empty — added X-Frame-Options (DENY), X-Content-Type-Options (nosniff), X-XSS-Protection, Referrer-Policy, Strict-Transport-Security (HSTS), and Permissions-Policy to all routes
- Set `productionBrowserSourceMaps: false` explicitly

**Weather API Route (`/api/weather/[raceId]/route.ts`)**
- Was fully unauthenticated and used the service role key — anyone on the internet could query it
- Added `authenticateRequest()` and ownership verification (race must belong to the requesting user)
- Removed `/api/weather` from the middleware public route exemptions
- Fixed error response to return generic message instead of raw Supabase error

**IDOR Fix (`/api/workouts/[id]/route.ts`)**
- The PATCH update query only filtered by workout ID, not user ID
- Added `.eq('user_id', user!.id)` to the update query as defense-in-depth

**Error Leakage (`/src/lib/api/utils.ts`, `/api/import/route.ts`, `/api/weather/route.ts`)**
- `handleApiError` was returning `err.message` to the client (could expose DB internals)
- Import route was returning Supabase `error.code`, `error.hint`, and `error.details` to the browser
- All routes now return "Internal server error" generically; full errors are logged server-side only

**Garmin Webhook (`/api/garmin/webhook/route.ts`)**
- Signature validation was wrapped in `if (process.env.GARMIN_WEBHOOK_SECRET && ...)` — skipped entirely if the env var was absent
- Made validation unconditional — returns 500 if secret is not configured, 401 if signature is invalid

**Garmin Auth Route (`/api/garmin/auth/route.ts`)**
- Had no explicit auth check — relied solely on middleware
- Added explicit `authenticateRequest()` call as defense-in-depth
- Callback URL was derived from the request Host header (SSRF/OAuth hijack risk)
- Changed to use `APP_URL` environment variable with fallback to request origin

**File Upload (`/api/import/route.ts`)**
- No server-side file size limit — client enforced 50MB but server accepted anything
- Added 50MB server-side check before any parsing

**Middleware Renamed**
- Next.js 16 deprecated `middleware.ts` — renamed to `proxy.ts`, function renamed from `middleware` to `proxy`
- Ran the official `@next/codemod` migration

**Environment Variables**
- Created `.env.example` documenting all required variables with notes on which are public vs private
- Added `APP_URL` env var to Vercel for hardcoded callback URLs
- `NEXT_PUBLIC_SUPABASE_ANON_KEY` was missing from Vercel — added to fix build failure
